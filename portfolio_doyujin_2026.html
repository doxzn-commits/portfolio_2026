수기 작성으로 정체되어 있던 내부 게시판을 AI 발행 파이프라인으로 전환하고, 같은 달·같은 채널 비교로 성과를 증명했습니다.

카피아 Content Growth GEO · AI 검색 최적화

CTA 클릭 16건 → 329건, 전환 0건 채널에서 상담 전환 4건 창출

내부 게시판(뉴스 · 차량 꿀팁) 콘텐츠 그로스 · 2026

Problem

담당자가 수기로 작성하던 게시판은 6개월간 CTA 클릭 90건, 전환 12건에 그칠 정도로 정체. 콘텐츠 품질 편차가 크고 발행량 확대가 구조적으로 불가능했습니다.

Hypothesis

"색인되는 글에는 공통 패턴이 있다. 이 패턴을 규칙화해 AI 생성 파이프라인에 강제하면, 발행량과 품질을 동시에 확보해 검색·AI 검색 유입과 전환을 만들 수 있다."

Execution

  • 원고 자동 생성 파이프라인 — CLI에 키워드를 입력하면 완성 HTML이 출력되는 구조 설계
  • GEO 표준 템플릿 — 정답 박스 · 질문형 H2 · FAQ · 자사 1차 비교표 · JSON-LD로 Google/AI 검색(AI Overview, Perplexity) 인용 가능성 극대화
  • 색인 규칙의 프롬프트화 — 색인 성공/실패 글을 비교 분석해 공통 패턴(제목 훅, 도입 하드데이터, 고유 앵글)을 생성 규칙으로 체계화
  • 개발 리소스 0 발행 — UI 재구축 없이 기존 에디터를 활용해 즉시 운영 시작
  • 전환 중심 설계 — 조회수가 아닌 CTA 클릭·상담 전환으로 측정, 모든 콘텐츠에 CTA 연결 강제
