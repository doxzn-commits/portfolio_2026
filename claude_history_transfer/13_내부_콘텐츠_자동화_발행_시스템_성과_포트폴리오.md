## 1. 프로젝트 개요

카피아 내부 게시판(뉴스·차량 꿀팁) 콘텐츠를 AI 기반으로 자동 생성·발행하는 시스템을 구축하고 운영했다.

이전에는 담당자가 콘텐츠를 수기로 작성해 발행했으며, 콘텐츠별 품질·구조 편차가 크고 발행량 확대가 어려웠다. 이를 대체하기 위해 다음을 하나의 발행 파이프라인으로 연결했다.

- CLI/Claude 기반 원고 자동 생성 (키워드 → 완성 HTML)
- GEO 표준 템플릿: 정답 박스, 질문형 H2, 정의형 직답, FAQ, 카피아 1차 비교표, body 내부 JSON-LD (BlogPosting·FAQPage·BreadcrumbList)
- 색인 규칙 프롬프트화: 제목 훅(숫자·순위·고유명사·연도 필수), 도입 하드데이터, 시점 고정, 고유 앵글
- 기존 raw HTML 에디터를 그대로 활용해 UI 재구축 없이 즉시 발행 (개발 리소스 0)
- 모든 콘텐츠는 상담 CTA로 연결되는 구조로 설계

핵심 관점: 단순 "글 자동 생성"이 아니라, 검색·AI 검색(GEO) 유입 → CTA 클릭 → 상담 전환으로 이어지는 콘텐츠 그로스 시스템.

## 2. 성과 비교 (2026년 6월, 동월·동일 채널)

같은 달(6월), 같은 내부 게시판에서 수기 작성 콘텐츠(old 링크)와 AI 자동화 발행 콘텐츠의 성과를 링크별로 비교했다.

| 구분 | 수기 작성 (AI 도입 전) | AI 자동화 발행 (도입 후) | 변화 |
|---|---|---|---|
| CTA 클릭 수 | 16건 | 329건 | **약 20배** |
| 상담 전환 수 | 0건 | 4건 | 0 → 4건 |
| 전환율 (전환/클릭) | 0% | 약 1.2% | — |

*클릭 수는 게시글 조회수가 아닌 CTA 버튼 클릭 수 기준.

### 참고 맥락

- 수기 작성 콘텐츠의 1~6월 누적 성과: 클릭 91건, 전환 6건 (월평균 클릭 약 15건). AI 도입 후 한 달 만에 이전 6개월 누적 클릭의 3.6배를 달성.
- 일별 추이: 6월 초 하루 1~8클릭 → 6월 말 하루 14~30클릭으로 발행 누적에 따라 우상향. 에버그린 구조 특성상 발행량이 쌓일수록 성과가 복리로 증가하는 패턴 확인.

## 3. 성과의 원인 분석

1. **발행량 확대**: 수기 작성 대비 콘텐츠 생산 병목이 제거되어 발행 규모 자체가 커졌다.
2. **구조 표준화**: 색인되는 글의 공통 패턴(제목 훅, 도입 직답, 수치 밀도)을 generation-prompt에 규칙화하여 콘텐츠 품질 편차를 제거했다.
3. **GEO 대응**: 정답 박스·FAQ·JSON-LD 구조로 Google 및 AI 검색 인용 가능성을 높였다.
4. **CTA 설계**: 모든 콘텐츠가 상담 CTA로 자연 연결되도록 템플릿 단계에서 강제하여, 트래픽이 아닌 클릭·전환 중심으로 측정했다.

## 4. 포트폴리오용 문장

한 줄 소개:
내부 게시판 콘텐츠를 AI 기반 자동 발행 시스템으로 전환하여, 동월 비교 기준 CTA 클릭을 16건에서 329건으로 약 20배 성장시키고 전환 0건 채널을 전환 발생 채널로 전환했다.

상세 문장:
수기 작성에 의존하던 카피아 내부 게시판 콘텐츠(뉴스·차량 꿀팁) 운영을 AI 기반 자동 발행 시스템으로 전환했다. CLI에 키워드를 입력하면 GEO 표준 구조(정답 박스, 질문형 H2, FAQ, JSON-LD)와 색인 규칙(제목 훅, 도입 하드데이터)이 반영된 완성 HTML이 생성되며, 기존 raw HTML 에디터로 즉시 발행되도록 설계해 개발 리소스 없이 운영을 시작했다. 그 결과 같은 달·같은 채널 비교에서 CTA 클릭이 16건 → 329건(약 20배)으로 성장했고, 6개월간 전환 6건에 그치던 채널에서 한 달 만에 상담 전환 4건이 발생했다. 이전 6개월 누적 클릭(91건)의 3.6배를 한 달 만에 달성했다.

이력서용 축약:
- 내부 게시판 콘텐츠 AI 자동 발행 시스템 구축 (원고 생성 → GEO 템플릿 → HTML 발행 파이프라인)
- 동월 비교 CTA 클릭 16건 → 329건 (약 20배), 상담 전환 0건 → 4건
- 이전 6개월 누적 클릭(91건) 대비 3.6배를 1개월 만에 달성
- 색인 성공 패턴을 프롬프트 규칙으로 체계화하여 콘텐츠 품질 표준화
- UI 재구축 없이 기존 에디터 활용 — 개발 리소스 0으로 발행 체계 전환

## 5. 면접 답변용 30초 소개

카피아 내부 게시판은 담당자가 수기로 콘텐츠를 작성하던 채널이었는데, 6개월간 CTA 클릭이 91건에 그칠 정도로 성과가 정체되어 있었습니다. 저는 이 채널을 AI 기반 자동 발행 시스템으로 전환했습니다. 키워드를 입력하면 GEO 구조와 색인 규칙이 반영된 완성 HTML이 나오도록 프롬프트와 템플릿을 설계했고, 기존 에디터를 그대로 활용해 개발 리소스 없이 바로 운영에 들어갔습니다. 그 결과 같은 6월 한 달 동월 비교에서 수기 콘텐츠는 클릭 16건·전환 0건이었는데, AI 발행 콘텐츠는 클릭 329건·전환 4건을 만들었습니다. 단순히 글을 자동으로 쓴 게 아니라, 색인되는 글의 패턴을 데이터로 찾아 규칙화하고 전환까지 이어지는 구조를 템플릿에 강제한 것이 핵심이라고 생각합니다.
