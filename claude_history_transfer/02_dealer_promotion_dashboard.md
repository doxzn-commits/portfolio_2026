# 딜러프로모션 대시보드 히스토리 인계서

원본 공유 링크: https://claude.ai/share/69dcffb1-fdab-4318-85df-40b007116fb3

## 1. 주제와 목표
딜러들이 카피아 담당자에게 전달하는 차량별 프로모션 가격을 취합하고, 최고 할인 조건을 프로모션 관리 및 협력업체 공유용 페이지에 연동하는 어드민 시스템 구축 이력이다.

최종 페이지:
1. 딜러 프로모션 취합
2. 프로모션 관리
3. 협력업체 공유용

프로모션 입력 유형:
- 자사 금융
- 타사 금융
- 현금

## 2. 핵심 상태 모델
초기에는 취합 저장과 홈페이지 반영이 한 단계로 묶일 위험이 있었다. 최종적으로 저장과 게시를 분리하는 3-Layer 모델로 정리했다.

Layer 1 입력/임시:
- 딜러 금액 입력 시 화면에서만 실시간 max 계산.
- DB와 홈페이지에는 아직 반영하지 않는다.

Layer 2 어드민 반영:
- 저장 버튼 클릭 시 프로모션 관리 페이지에 반영.
- 홈페이지에는 아직 노출하지 않는다.

Layer 3 홈페이지 게시:
- 차량노출 토글과 저장 및 게시 버튼으로 확정.
- publishedAmount 같은 게시 시점 스냅샷을 별도로 둔다.

핵심 원칙:
- 입력 즉시 화면 계산은 가능하다.
- 저장 전에는 협력업체 공유용에 반영되면 안 된다.
- 어드민 최신값과 홈페이지 게시값은 분리한다.

## 3. 딜러 프로모션 취합 페이지
1440px PC 화면 기준.

좌측 패널:
- 브랜드
- 차종
- 모델명
- 세부트림
- 연식
- 차량가격
- 자사 max
- 타사 max
- 현금 max

우측 패널:
- 딜러명별 컬럼
- 각 딜러 아래 자사, 타사, 현금 입력칸
- 딜러는 계속 추가 가능
- 모든 금액은 수기 입력

최종 입력 규칙:
- 모든 금액 input은 controlled input이어야 한다.
- value와 onChange를 함께 사용한다.
- 입력 즉시 calculateMaxPerType이 재계산되어야 한다.
- 초록 배경은 focus CSS일 뿐 로직과 분리한다.

max 표시:
- 금액 아래에 해당 딜러명을 작은 폰트로 표시한다.
- 같은 유형 안에서 최고가가 동률이면 딜러명을 콤마로 모두 표시한다.
- 긴 딜러명 목록은 말줄임과 tooltip으로 처리한다.

유형별 max와 대표 max:
- ownMaxAmount, ownMaxDealer
- otherMaxAmount, otherMaxDealer
- cashMaxAmount, cashMaxDealer
- 대표 최고값은 별도로 계산한다.
- 최종 프로모션 관리에서는 자사/타사/현금 중 제일 높은 1개 금액만 보여준다.

필터와 검색:
- 좌측 패널에는 브랜드 필터, 차종 필터, 차량명 검색, 초기화, 결과 카운트가 있다.
- 우측 패널에는 딜러명 검색과 딜러 추가 버튼이 있다.
- 필터는 화면 표시만 줄인다. 필터로 숨겨진 차량이나 딜러 데이터가 사라지면 안 된다.

좌우 y좌표 정렬:
- 좌측 thead가 1행이고 우측 thead가 2행이라 tbody 시작 위치가 어긋났다.
- 해결은 좌측에도 더미 헤더 행을 추가해 양쪽 thead를 2행으로 맞추는 것이다.
- ROW_H = 46 같은 고정 행 높이를 사용한다.

차량 추가와 우측 셀 생성:
- 차량 추가 후 우측 딜러 입력 셀이 생기지 않는 문제가 있었다.
- 원인은 key 없는 빈 Fragment로 td 그룹을 감싼 것이다.
- React.Fragment에 key를 부여해야 한다.

## 4. 엑셀 업로드 기능
딜러별 엑셀 파일을 업로드하면 자동으로 차량/딜러 데이터가 입력되어야 한다.

엑셀 구조:
- 브랜드별 시트.
- 모든 시트가 동일 구조.

사용 컬럼:
- B 시리즈
- C 연식
- D 모델명
- E 코드명/트림
- H 보유댓수
- I 차량가. 원 단위이므로 만원으로 변환.
- K 즉출가능성
- L 최신월 자사. 원 단위이므로 만원으로 변환.
- M 최신월 타사. 원 단위이므로 만원으로 변환.
- N 현금. 원 단위이므로 만원으로 변환.

최신월 탐지:
- L 컬럼 헤더의 XX년 XX월 자사 패턴으로 최신월 자사/타사/현금 컬럼을 찾는다.

업로드 흐름:
1. 엑셀 업로드 버튼 클릭
2. 파일 선택
3. 미리보기 모달
4. 딜러 지점명 입력 또는 파일명에서 자동 추출
5. N개 차량 데이터 반영 클릭

반영 규칙:
- 브랜드 + 모델 + 연식이 같으면 기존 차량의 딜러 입력값만 업데이트.
- 신규 차량이면 차량 목록에 추가.
- 딜러가 없으면 새 딜러 컬럼도 추가.

중요 버그:
- JSX 안의 script 태그는 React에서 실행되지 않는다.
- SheetJS는 useEffect로 동적으로 로드해야 한다.
- 로드 전에는 업로드 버튼을 비활성화한다.

## 5. 프로모션 관리 페이지
최종 수정사항:
- 할인유형 컬럼 제거.
- 최고 할인금액은 자사/타사/현금 중 가장 높은 단일 값만 노출.
- 금액 형식은 2,493만원.
- 자사/타사 안내 텍스트는 금액 컬럼에 노출하지 않는다.
- 가격게시 컬럼 삭제.
- 저장 버튼과 차량노출 토글로 게시 흐름을 통합.

저장/게시 UX:
- 차량노출 OFF + 저장: 변경사항만 저장, 미게시 유지.
- 차량노출 ON + 미게시 + 저장: 저장 및 게시. 확인 모달 필요.
- 차량노출 ON + 게시중: 게시중 상태 표시.
- 게시중에서 토글 OFF: 게시 취소 확인 모달. 확정 시 isPublished=false, isVisible=false.
- 게시 취소 링크는 삭제한다.
- 저장 버튼 하단에 게시 일자를 표시한다.

이력 관리:
- 기존 PROMOTION_HISTORIES 상수는 하드코딩이라 최신값이 반영되지 않았다.
- 최종 구조는 각 promotion 객체 안에 histories 배열을 둔다.
- 취합 탭 저장 후 최고 할인금액이 바뀌면 histories 맨 앞에 새 이력 행을 추가한다.
- 이력 모달에서 수정한 내용도 promotions 상태에 반영한다.

이력 상세 최종 컬럼:
- 등록일시
- 자사 할인
- 타사 할인
- 제안딜러
- 관리

주의:
- 현금 할인 컬럼은 삭제됐다.
- 타사 할인은 내부적으로 타사와 현금 중 큰 값을 자동 반영한다.
- 모든 값은 수정 가능해야 한다.

## 6. 협력업체 공유용 페이지
세 번째 페이지로 신설됐다.

역할:
- 협력업체에 공유할 수 있는 VIEW ONLY 페이지.
- 딜러별 제안가는 공개하지 않는다.
- 딜러 프로모션 취합의 좌측 패널과 유사하되 딜러명을 숨긴다.

구성:
- 사이드바 메뉴에 협력업체 공유용 추가.
- VIEW ONLY 뱃지.
- 마지막 업데이트 일시와 기준 월 표시.
- 금액만 표시하는 MaxCellPartner.
- 필터와 검색 제공.

중요 연동 수정:
- 처음에는 PartnerPage가 dealerState.maxResults를 직접 참조해 저장 전에도 실시간 반영됐다.
- 최종적으로 App 루트에 savedSnapshot을 만들었다.
- PartnerPage는 savedSnapshot만 참조한다.
- 저장 버튼 클릭 시에만 savedSnapshot이 갱신된다.

## 7. ver 1 안정화 이력
딜러 프로모션 ver 1에서 수정된 주요 항목:
- 탭 전환 시 입력 데이터 소실 방지. state를 App 루트로 올림.
- Mercedes-Benz와 벤츠 매핑 실패 방지. BRAND_NORMALIZE 사용.
- 저장 중 재클릭 중복 저장 방지.
- savedMax 보존.
- 차량 ID별 이력 분리.
- HistoryModal 분리와 인라인 수정 모드.
- 페이지 변경 시 체크박스 선택 초기화.
- 게시불일치 모달의 publishedAmount 동적 비교.

## 8. 기능정의서와 PPT
최종 문서 범위는 3페이지 전체다.
- 딜러 프로모션 취합
- 프로모션 관리
- 협력업체 공유용

요구사항:
- 중학생도 이해할 수 있는 수준.
- Pretendard 폰트 고정.
- 페이지별 버튼, 트리거, 데이터 송수신, 연동 시나리오를 보여준다.

최종 기능정의서 예시:
- 기능 정의서 전체 29개 내외 항목.
- 연동 시나리오 표 11가지.
- 개발 체크리스트 20개 내외.

## 9. 다음 AI 주의사항
- Figma Make가 완료했다고 해도 JSX 렌더링이 실제로 바뀌지 않는 경우가 있었다. 중요한 로직은 직접 코드로 작성하라.
- max 계산은 controlled input + onChange가 필수다.
- 협력업체 공유용은 저장 전 실시간 연동 금지다.
- 이력은 하드코딩 상수가 아니라 상태 내부에서 관리해야 한다.
- 최종 프로모션 관리 최고 할인금액은 단일 최고값만 보여준다.
- 저장과 게시는 UI상 토글과 저장 버튼으로 직관화한다.
