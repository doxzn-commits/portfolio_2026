# 차량 꿀팁 게시판 원고 및 GEO 템플릿 히스토리 인계서

원본 공유 링크: https://claude.ai/share/dca7a17e-00ca-4fc8-94ec-24deca173114

## 1. 주제와 목표
카피아 대시보드의 차량 꿀팁 게시판 원고를 자동 생성하고, Google 검색과 AI 검색 결과에서 카피아가 인용될 수 있도록 GEO 구조를 반영하는 템플릿/프롬프트/서버 핸드오프 스펙을 정리한 이력이다.

목표:
- 구매 꿀팁, 차량 관리 꿀팁, 법률 꿀팁 등 에버그린 콘텐츠 자동 생성.
- Google AI Overview, Perplexity, ChatGPT 검색형 답변 등에 인용될 가능성 확대.
- 기존 뉴스 콘텐츠 생성 흐름처럼 CLI에 키워드를 주면 원고를 생성.
- 대시보드 UI를 크게 재구축하지 않고 기존 raw HTML 에디터를 우선 활용.

## 2. 기존 원고 문제 진단
주요 문제:
- localhost 이미지 URL이 JSON-LD, og:image, 본문 이미지 등에 남아 있었다.
- 크롤러는 localhost에 접근할 수 없으므로 production URL로 치환해야 한다.
- 꿀팁 콘텐츠인데 NewsArticle 스키마를 사용했다.
- 꿀팁은 BlogPosting 또는 Article, FAQPage, 절차형이면 HowTo가 맞다.
- H2 첫 문장의 직답성이 약했다.
- 공식 출처와 수치가 부족했다.
- 카피아만의 1차 데이터, 예를 들어 구매 vs 리스 vs 장기렌트 비교표가 부족했다.

## 3. 최종 구축 범위
최종 선택은 템플릿 + 스펙만, UI 변경 0이다.

이유:
- 기존 에디터가 이미 raw HTML을 받을 수 있다.
- Claude/CLI가 완성 HTML을 바로 출력하면 렌더 엔진을 새로 만들 필요가 낮다.
- 서버 head/meta 문제가 해결되기 전에는 대시보드 UI 재구축 ROI가 낮다.
- body 안 JSON-LD script는 head 제거를 우회할 수 있다.

권장 산출물 폴더:
- docs/tips-geo-template/

산출물 4종:
1. template.html
2. generation-prompt.md
3. sample-구매-신차세금.html
4. SERVER-HANDOFF.md

## 4. 표준 콘텐츠 구조
차량 꿀팁 표준 구조:
1. 정답 박스. 맨 위 40-60자 직답.
2. 도입. 첫 문장은 검증 가능한 직답.
3. 목차.
4. 질문형 H2.
5. H2 첫 문장 정의형 직답.
6. 표 또는 체크리스트.
7. 섹션별 한 줄 결론.
8. 카피아 고유 비교표.
9. 핵심 요약 박스.
10. FAQ.
11. CTA.
12. body 내부 JSON-LD.

스키마:
- BlogPosting
- FAQPage
- BreadcrumbList
- 절차형 글이면 HowTo 옵션

NewsArticle은 기본값으로 쓰지 않는다.

## 5. 서버 핸드오프 핵심
즉시 효과:
- JSON-LD를 body 안에 넣으면 extractBodyInner가 head를 제거해도 보존될 수 있다.
- Google은 body 내 JSON-LD도 인식할 수 있다.
- 따라서 개발자는 발행 전처리가 body 내부 ld+json script를 보존하는지만 우선 확인하면 된다.

개발자 작업:
- /tips/{slug} 운영 라우트 매핑.
- /board/post/{cuid} 같은 내부 ID URL을 키워드 기반 slug로 개선.
- title, meta description, canonical, og, dateModified를 운영 템플릿에서 공급.
- sitemap lastmod는 실제 updatedAt 기준으로 수정.
- 홈/카테고리 허브에 최신/추천 글 내부 링크 모듈 추가 검토.

## 6. 색인 이슈 분석 결론
특정 뉴스 글이 GSC URL 검사에서는 보이지만 제목 검색이나 site 검색에서 안 보이는 문제가 있었다.

초기 가설:
- 미색인
- CSR 렌더링 문제
- canonical 누락
- sitemap 누락

이후 정정:
- 스크린샷과 HTML이 정상인 경우 CSR 문제는 폐기.
- 일부 케이스는 크롤링됨 - 현재 색인이 생성되지 않음 상태였다.
- 이는 구글이 페이지를 봤지만 색인 우선순위나 독자적 가치를 충분히 느끼지 못한 상태로 해석했다.
- 실패 글은 오히려 길고 구조도 많아 thin content만의 문제는 아니었다.

사용자는 과도한 사이트 전체 구조 문제 해석보다 걸리는 글을 만드는 데 집중하자고 했다.

최종 방향:
- 이미 색인된 글의 공통 패턴을 템플릿에 반영한다.
- 색인 수사를 지나치게 키우지 않는다.

## 7. 색인되는 글의 특징 규칙
generation-prompt.md에 반영한 핵심 규칙:

우선순위:
- 최우선: 제목 훅, 도입 하드데이터, 시점 고정.
- 강: 고유 앵글.
- 보조: 수치 밀도.

제목 규칙:
- 숫자, 순위, 고유명사, 연도 중 최소 1개 필수.

금지할 소프트 제목:
- ~경쟁 총정리
- ~시대
- ~알아보기
- ~란?
- ~이야기

도입 첫 문장:
- 정성 서사 금지.
- 검증 가능한 직답으로 시작.

뉴스형:
- 제목 훅, 도입 하드데이터, 수치 밀도, 시점 고정, 고유 앵글.

꿀팁형:
- 제목 훅, 도입 하드데이터, 수치 밀도, 고유 앵글.
- 시점 고정 대신 연도 명시 + 카피아 1차 비교 데이터.

## 8. 최종 마무리 지시
- generation-prompt.md의 규칙이 단일 진실 공급원이다.
- README에는 색인 우선순위 규칙은 generation-prompt.md §4.5 참조 라는 포인터 한 줄만 둔다.
- template.html에는 색인 규칙을 복제하지 않는다.
- generation-prompt.md 출력 스키마와 template.html placeholder가 1:1 일치하는지 확인한다.
- 샘플 HTML이 새 제목/도입 규칙을 지키는지 확인한다.

## 10. 다음 AI 주의사항
- 차량 꿀팁은 뉴스와 다르게 에버그린이다.
- NewsArticle을 기본 스키마로 쓰지 않는다.
- raw HTML 에디터 유지 결정이 있었다. UI 재구축부터 시작하지 말 것.
- 색인 이슈는 필요 이상으로 크게 확장하지 말 것.
- 꿀팁형에는 연도판본과 카피아 1차 비교 데이터를 넣는다.
