## 1. 프로젝트 개요

SEO/GEO 진단기는 URL을 입력하면 웹사이트의 검색 최적화 상태와 AI 인용 최적화 상태를 함께 진단하는 내부 분석 도구다.

일반적인 SEO 분석이 검색엔진 노출을 위한 기술 점검에 머무르는 반면, 이 프로젝트는 AI 검색 환경에서 중요한 GEO, 즉 Generative Engine Optimization 관점을 별도 점수 체계로 분리했다. 단순히 검색 결과에 노출되는지뿐 아니라 AI가 답변을 만들 때 인용하기 쉬운 구조인지, 질문 의도에 직접 답하는 문장이 있는지, FAQ와 정의형 문장, 구조화 데이터가 갖춰져 있는지를 함께 측정하도록 설계했다.

도구는 CLI와 웹 UI를 모두 지원한다. CLI에서는 URL과 타깃 질문을 입력해 JSON, HTML, Markdown 개선 리포트를 생성할 수 있고, Express 서버 기반 UI에서는 브라우저에서 URL을 넣고 진단 결과와 개선 리포트를 확인할 수 있다.

---

## 2. 해결한 업무 문제

마케팅 관점에서 SEO 개선 업무는 외부 도구 결과를 사람이 해석하고, 페이지별 개선 우선순위를 수동으로 정리해야 하는 경우가 많다. 특히 AI 검색 노출을 목표로 할 때는 기존 SEO 점수만으로는 부족했다.

이 프로젝트는 다음 문제를 해결하기 위해 만들어졌다.

- 기술 SEO, 콘텐츠 품질, AI 인용 가능성, 답변 구조를 한 번에 비교하기 어려움
- URL별 문제점을 사람이 직접 해석해야 하는 반복 업무 발생
- 어떤 항목부터 개선해야 점수 상승 효과가 큰지 판단하기 어려움
- AI 답변에 인용되기 위한 FAQ, 정의형 문장, 구조화 데이터 기준이 내부적으로 정리되어 있지 않음
- 클라이언트 또는 내부 보고용 리포트를 매번 수동 작성해야 함

이를 URL 진단, 점수화, 등급화, 개선 우선순위 도출, 리포트 생성까지 한 흐름으로 자동화했다.

## 3. 주요 기능

### SEO 점수화

SEO 점수는 100점 만점으로 구성된다.

| 영역 | 비중 | 주요 측정 항목 |
|---|---:|---|
| 기술적 SEO | 25% | Core Web Vitals, robots.txt, sitemap, HTTPS, 모바일 최적화, 구조화 데이터 |
| 콘텐츠 품질 | 30% | 본문 길이, 키워드 구조, H1/H2 구성, 콘텐츠 깊이 |
| 온페이지 최적화 | 20% | 타이틀, 메타디스크립션, URL 구조, 내부 링크 |
| 백링크 권위 | 15% | 도메인 권위, 외부 권위 신호 |
| 사용자 경험 | 10% | PageSpeed 기반 UX 점수 |

API 키가 없는 항목은 Mock 점수로 처리해 도구가 중단되지 않도록 설계했다. 이를 통해 개발 초기에도 전체 진단 흐름을 검증할 수 있었다.

### GEO 점수화

GEO 점수는 AI 답변에 인용될 가능성을 측정하는 별도 100점 체계다.

| 영역 | 비중 | 주요 측정 항목 |
|---|---:|---|
| 인용 가능성 | 25% | 정의형 문장, 수치·통계 포함 여부 |
| 권위·신뢰성 | 25% | 저자, 날짜, 출처, 브랜드 신호 |
| AI 친화 구조 | 20% | 헤딩 계층, FAQ 구조, 명확한 문장 |
| 답변 일치도 | 20% | 타깃 질문과 본문 구조의 일치 여부 |
| 멀티소스 노출 | 10% | OG/Twitter Card 등 외부 노출 메타 |

타깃 질문 옵션을 통해 특정 질문을 넣으면, 해당 질문이 헤딩과 첫 문단 등 주요 위치에 반영되어 있는지도 분석한다.

### 리포트 생성

지원 리포트:
- 콘솔 리포트
- JSON 리포트
- HTML 리포트
- Markdown 개선 리포트
- HTML 개선 리포트

리포트에는 SEO/GEO 총점, 등급, 영역별 획득 점수, 항목별 상태, Mock 처리 여부, 타깃 질문 목록, 개선 우선순위, 쉬운 개선과 개발 필요 개선 구분, 예상 점수 상승 효과가 포함된다.

### 웹 UI 및 API

Express 기반 서버를 통해 웹 UI와 API를 제공한다.

주요 API:
- health check
- 측정 요청
- 스트리밍 측정 진행률
- output 리포트 파일 조회

이를 통해 개발자용 CLI뿐 아니라 비개발자도 브라우저에서 URL을 입력해 진단 결과를 확인할 수 있도록 했다.

## 5. 검증 및 QA

WORK.md 기준으로 다음 항목이 검증되었다.

- SEO/GEO 총점이 0~100 사이로 출력되는지 확인
- 영역별 점수 합산이 총점과 일치하는지 확인
- Mock 처리 항목 표시 확인
- 등급 구간 확인
- robots.txt, 타이틀/메타 길이, 콘텐츠 길이, 구조화 데이터 파싱 확인
- 한국어 정의형 문장 패턴 탐지 확인
- 숫자·통계 패턴 탐지 확인
- 타깃 질문 옵션 반영 확인
- HTML/JSON 리포트 저장 확인
- 존재하지 않는 URL에서도 프로세스가 중단되지 않도록 오류 처리 개선

샘플 결과로 카피아, 핌아시아, example.com, naver.com 등의 HTML/JSON/개선 리포트가 output 폴더에 생성되어 있다.
