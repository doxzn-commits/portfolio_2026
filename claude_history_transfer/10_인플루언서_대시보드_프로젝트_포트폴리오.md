## 1. 프로젝트 개요

인플루언서 대시보드는 카피아의 블로그 인플루언서 마케팅 운영 데이터를 통합 관리하기 위한 Google Apps Script + HTML 기반 사내 자동화 시스템이다.

프로젝트는 크게 두 축으로 구성된다.

1. 인플루언서별 정산 시트를 월 단위로 통합하고 대시보드에서 성과를 시각화하는 정산 대시보드
2. 원본 상담 DB를 레퍼러 ID 기준으로 각 인플루언서의 개별 시트에 자동 분배하는 DB 분배 시스템

이를 통해 인플루언서 성과 확인, 정산 대상 확인, 상담 DB 분배, 월별 전환 집계를 자동화했다.

---

## 2. 해결한 업무 문제

기존 인플루언서 운영은 개별 구글 시트와 수기 정산에 의존했다.

주요 문제:
- 인플루언서별 데이터가 분산되어 전체 성과를 한눈에 보기 어려움
- 월별 상담 DB 수와 정산금액을 수동으로 집계해야 함
- 정산일자와 판매완료 여부처럼 사람이 입력한 값이 자동화로 덮어써질 위험이 있음
- 신규 상담 DB를 레퍼러 ID 기준으로 인플루언서별 시트에 나눠주는 작업이 반복적으로 필요함
- 운영팀이 누가 정산 대기인지, 누가 성과가 좋은지, 어떤 DB가 어느 인플루언서에게 배정됐는지 확인하는 데 시간이 걸림

이 프로젝트는 Google Workspace 안에서 별도 서버 없이 자동 동기화와 시각화 대시보드를 구현해 월간 운영 시간을 크게 줄이도록 설계되었다.

---

## 3. 정산 통합 자동화

핵심 파일:
syncInfluencers.gs

주요 역할:
- Google Drive 폴더 안의 인플루언서별 Google Sheets를 순회한다.
- 파일명에서 블로그명과 실명을 파싱한다.
- 각 시트의 일별 데이터를 월별로 집계한다.
- 상담 DB 수, 전환율, 정산금액을 자동 계산한다.
- 마스터 통합 시트에 월 + 인플루언서 기준으로 upsert한다.
- 정산일자는 사람이 입력한 값이므로 자동화가 덮어쓰지 않는다.
- 매일 새벽 4시 시간 트리거로 자동 실행할 수 있다.

정산 계산:
- 전환율 = 상담DB / 클릭수
- 정산금액 = 상담DB x 10,000원

마스터 시트 주요 컬럼:
- 진행월
- 인플루언서명
- 블로그명
- 클릭수
- 상담DB
- 전환율
- 판매완료
- 정산일자
- 정산금액

중요한 설계 결정:
- append-only가 아니라 월 + 인플루언서 기준 upsert를 사용한다.
- 사람이 입력하는 정산일자는 보존한다.
- 자동 계산 컬럼과 수기 입력 컬럼을 명확히 분리한다.

## 5. 상담 DB 자동 분배 시스템

핵심 파일:
dbDistributor.gs

주요 역할:
- 원본 통합 상담 DB 시트를 읽는다.
- 문의경로 또는 레퍼러 정보에서 레퍼러 ID를 추출한다.
- Config 탭에서 레퍼러 ID와 인플루언서별 개별 시트 정보를 매칭한다.
- 각 인플루언서의 개별 시트에 DB를 자동 분배한다.
- 연락처는 마스킹한다.
- 월 + 전화번호 기준 중복키를 생성해 중복 분배를 방지한다.
- 12시간 간격 트리거로 최근 DB를 동기화한다.
- bootstrap, backfill, recalc, monitoring, debug 함수를 제공한다.

출력 컬럼:
- 날짜
- 성함
- 연락처
- 구매방법
- 문의 모델
- 레퍼러 ID
- 문의 경로
- 중복키

운영 관련 기능:
- 초기 대량 데이터 bootstrap
- 최근 데이터 동기화
- 특정 인플루언서 백필
- 전체 통계 재계산
- Config 미매칭 레퍼러 디버깅
- 날짜 행 자동 확장
- 모니터링 시트 생성

## 6. 주요 기술적 의사결정

Google Apps Script 기반 구현:
이미 Google Sheets 중심으로 업무가 운영되고 있었기 때문에 별도 DB·서버·인증 시스템을 추가하지 않고 Google Workspace 안에서 자동화하는 것이 운영 비용과 유지보수 측면에서 유리했다.

단일 HTML 대시보드:
React/Vue 같은 프레임워크 대신 단일 HTML + Vanilla JS 구조를 선택했다. 마케팅 운영자가 별도 설치 없이 파일을 열어 사용할 수 있어야 했기 때문이다.

JSONP 기반 Google Sheets 읽기:
로컬 HTML 파일에서 fetch를 사용하면 CORS 문제가 발생할 수 있다. 이를 피하기 위해 Google Sheets gviz JSONP API를 사용해 HTML 파일 더블클릭만으로 데이터를 읽을 수 있게 구성했다.

수기 입력값 보존:
정산일자와 판매완료처럼 사람이 판단해서 넣는 값은 자동화가 덮어쓰지 않도록 분리했다. 이 결정은 자동화 도입 이후에도 운영팀의 수기 판단 데이터가 사라지지 않게 하는 핵심 안전장치다.

## 7. 업무적 성과

- 분산된 인플루언서별 시트를 하나의 마스터 시트와 HTML 대시보드로 통합했다.
- 월별 정산 집계와 성과 확인에 들어가던 반복 작업을 크게 줄였다.
- 정산 대기 상태와 인플루언서별 성과를 즉시 확인할 수 있는 운영 화면을 만들었다.
- 상담 DB 분배, 중복 방지, 연락처 마스킹, 전환 통계 재계산까지 자동화했다.
- 별도 서버 없이 Google Workspace만으로 동작하는 저비용 자동화 체계를 구축했다.

## 8. 포트폴리오용 문장

한 줄 소개:
분산된 인플루언서 정산 시트와 상담 DB를 Google Apps Script 기반으로 자동 통합·분배하고, 단일 HTML 대시보드로 성과와 정산 상태를 실시간에 가깝게 확인하는 운영 시스템을 구축했다.

상세 문장:
카피아의 블로그 인플루언서 마케팅 운영 과정에서 발생하던 정산 집계, 상담 DB 분배, 성과 확인 업무를 Google Apps Script와 단일 HTML 대시보드로 자동화했다. 인플루언서별 개별 시트를 월 단위로 집계해 상담 DB, 전환율, 정산금액을 자동 계산하고, 운영자가 수기 입력하는 정산일자는 보존하도록 설계했다. 또한 원본 상담 DB를 레퍼러 ID 기준으로 인플루언서별 시트에 자동 분배하고, 연락처 마스킹·중복 방지·통계 재계산 기능을 구현해 월간 운영 리드타임과 정산 누락 리스크를 크게 줄였다.

이력서용 축약:
- Google Apps Script 기반 인플루언서 정산 자동화 및 DB 분배 시스템 구축
- 월 + 인플루언서 기준 upsert, 정산일자 보존, 전환율/정산금액 자동 계산 구현
- 레퍼러 ID 기반 상담 DB 자동 분배, 연락처 마스킹, 중복 방지 로직 구현
- 단일 HTML 대시보드로 KPI, 월별 추이, 랭킹, 정산 대기 현황 시각화
- 별도 서버 없이 Google Workspace 기반으로 월간 반복 운영 업무 대폭 절감
